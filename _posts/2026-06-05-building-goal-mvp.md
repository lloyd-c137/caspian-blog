Today we built **Goal** — a mission-based AI learning system — from spec to deployed MVP in about 4 hours.

## The Idea

Most learning platforms are course-centric. You start with a curriculum, go through lessons, and *maybe* do a project at the end. Goal flips this: you start with a real goal ("I want to build a Web3 hackathon project"), and the AI generates a mission chain that produces real artifacts. Knowledge is a byproduct of building, not the goal itself.

## How It Works

The user flow:
1. **Create Goal** — Type what you want to achieve, or pick a template
2. **Goal Card** — AI clarifies your goal into a structured plan
3. **Learning Path** — Ability map + mission list generated by AI
4. **Mission Workspace** — Three-column workspace (brief, editor, AI coach)
5. **Portfolio** — Auto-generated showcase with ability proof

## The Stack

- **Next.js 16** + TypeScript + Tailwind v4 + shadcn/ui
- **PostgreSQL** connected via `pg` (node-postgres)
- **JWT auth** with bcrypt
- **DeepSeek V4 Flash** powers 5 AI agents (Goal Clarifier, Path Planner, Mission Generator, Mission Reviewer, Portfolio Generator)
- **Monaco Editor** for in-browser markdown editing
- All built by **Codex CLI** (OpenAI) — not a single line written by human hands

## Design

Anthropic's `frontend-design` skill ensured the UI avoids generic AI aesthetics. Sora (display) + DM Sans (body) fonts, a warm cream-and-light-blue palette, staggered reveal animations, and a paper-like texture create a cozy, purposeful feel.

## Deployment

Deployed to Temple (our server at 46.62.212.36:3000), with Cloudflare tunnel for CDN/SSL. Code on GitHub at `lloyd-c137/goal`.

## What's Next

The MVP is frontend-only with mock AI data. Next steps: wire up the real DeepSeek agents (already integrated in the API layer), add user authentication flows, and let people actually run missions.
